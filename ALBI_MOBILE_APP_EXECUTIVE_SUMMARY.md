# 📱 Albi 모바일 앱 개발 프로젝트 - 프로젝트 계획서 (Executive Summary)

## 📋 프로젝트 개요

### 프로젝트명
**Albi 모바일 앱 (React Native + Toss Apps-in-Toss 동시 출시)**

### 프로젝트 목표
현재 운영 중인 Albi 웹 애플리케이션(https://albi.kr)을 React Native 기반 모바일 앱으로 전환하여 다음 3개 플랫폼에 동시 출시:
1. **토스 앱인토스** (무료, 3,000만 사용자)
2. **Google Play Store** ($25 일회성)
3. **Apple App Store** ($99/년)

### 비즈니스 가치
- ✅ **무료 배포**: 토스 앱인토스를 통해 무료로 3,000만 사용자에게 노출
- ✅ **앱스토어 진출**: Google Play, App Store에서 공식 앱으로 다운로드 가능
- ✅ **기존 코드 재사용**: 웹앱 90% 재사용 (React Native WebView + 네이티브 기능)
- ✅ **빠른 개발**: 6-10주 내 3개 플랫폼 동시 출시
- ✅ **낮은 비용**: 첫 해 약 $124 (₩169,000)

## 🎯 전략적 접근: React Native Hybrid 아키텍처

### 왜 React Native Hybrid인가?

#### ✅ 장점
1. **기존 웹앱 재사용** (90% 코드 보존)
   - 웹앱은 이미 완성되어 있음 (https://albi.kr)
   - WebView로 기존 기능 모두 로드 가능
   - API 엔드포인트 그대로 사용

2. **3개 플랫폼 동시 지원**
   - Toss Apps-in-Toss (React Native + Granite SDK)
   - Android (Google Play Store)
   - iOS (Apple App Store)
   
3. **네이티브 기능 통합**
   - 푸시 알림 (Firebase FCM/APNs)
   - 생체 인증 (지문, Face ID)
   - 카메라 접근
   - 로컬 스토리지
   - 네이티브 UI 컴포넌트

4. **앱스토어 승인 보장**
   - Apple 4.2 정책 충족 (WebView만 사용 시 거절 위험)
   - Google Play 네이티브 기능 요구사항 충족

#### ⚠️ 대안 비교

| 전략 | Toss 앱인토스 | Google Play | App Store | 개발 시간 | 유지보수 | 추천도 |
|------|--------------|-------------|-----------|----------|---------|--------|
| **WebView 전용** | ✅ | ⚠️ 거절 위험 | ⚠️ 거절 위험 | 1-2주 | ★★★★★ | ★★ |
| **React Native Hybrid** | ✅ | ✅ | ✅ | 6-10주 | ★★★★ | ★★★★★ |
| **Full Native** | ❌ 미지원 | ✅ | ✅ | 3-6개월 | ★★ | ★★ |

**결론**: React Native Hybrid가 유일한 최적 솔루션
